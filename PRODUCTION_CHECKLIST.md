# Production Readiness & Optimization Report

Generated by full codebase review. Fix in priority order before deploying to production.

---

## CRITICAL — Fix Before Any Production Deployment

### 1. Hardcoded SECRET_KEY (SECURITY)
**File:** `backend/backend_project/settings.py`
**Risk:** The secret key is committed to GitHub. Anyone can forge JWT tokens, session cookies, and CSRF tokens.
**Fix:**
```python
import os
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY')
# Generate one with: python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

---

### 2. DEBUG = True in Production (SECURITY)
**File:** `backend/backend_project/settings.py`
**Risk:** Full stack traces, SQL queries, internal file paths exposed to anyone who triggers an error.
**Fix:**
```python
DEBUG = os.environ.get('DJANGO_DEBUG', 'False') == 'True'
```

---

### 3. CORS_ALLOW_ALL_ORIGINS = True (SECURITY)
**File:** `backend/backend_project/settings.py`
**Risk:** Any website can make authenticated cross-origin requests to the API.
**Fix:**
```python
CORS_ALLOW_ALL_ORIGINS = False
CORS_ALLOWED_ORIGINS = [
    'https://yourdomain.com',
    'http://localhost:5173',  # dev only
]
```

---

### 4. No Rate Limiting on Auth Endpoints (SECURITY)
**File:** `backend/backend_project/urls.py` — `/api/token/`
**Risk:** Unlimited brute-force password attempts. No lockout after failed logins.
**Fix:** Install `django-ratelimit` or `djangorestframework-simplejwt` throttling:
```python
# settings.py
REST_FRAMEWORK = {
    'DEFAULT_THROTTLE_CLASSES': ['rest_framework.throttling.AnonRateThrottle'],
    'DEFAULT_THROTTLE_RATES': {'anon': '10/min'},
}
```

---

### 5. No File Size Validation Per Upload (SECURITY / STABILITY)
**File:** `backend/resources/views.py`, `backend/events/views.py`
**Risk:** A single user can upload a 500MB file and exhaust DB storage or crash the server. The global limit exists in settings but is not enforced per-user or per-resource.
**Fix:** Add per-upload size check in `create()` before saving:
```python
file = request.FILES.get('file_data')
if file and file.size > 50 * 1024 * 1024:  # 50MB per file
    return Response({'error': 'File too large. Max 50MB.'}, status=400)
```

---

### 6. Profile Pictures in Every User List Response (PERFORMANCE / SECURITY)
**File:** `backend/users/serializers.py` — `UserSerializer.get_profile_picture_data()`
**Risk:** `/api/users/` returns all users with full base64-encoded profile pictures. With 100 users each having a 200KB photo, that's 20MB+ per request.
**Fix:** Exclude `profile_picture_data` from list serializer, only include it in detail/me endpoints:
```python
class UserListSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ['id', 'username', 'email', 'first_name', 'last_name', 'role', 'staff_title', 'department']
```

---

### 7. AUTH_COOKIE_SECURE = False (SECURITY)
**File:** `backend/backend_project/settings.py`
**Risk:** JWT cookies sent over plain HTTP — vulnerable to interception.
**Fix:**
```python
'AUTH_COOKIE_SECURE': os.environ.get('COOKIE_SECURE', 'False') == 'True',
```

---

### 8. ALLOWED_HOSTS = [] (SECURITY)
**File:** `backend/backend_project/settings.py`
**Risk:** Django accepts requests from any host header — HTTP Host header injection.
**Fix:**
```python
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', 'localhost,127.0.0.1').split(',')
```

## IMPORTANT — Fix Before Going Live

### 9. Hardcoded localhost URL in AuthContext (DEPLOYMENT)
**File:** `src/contexts/AuthContext.tsx` line ~50
**Risk:** `fetch('http://127.0.0.1:8000/api/auth/me/', ...)` is hardcoded — will fail in production.
**Fix:** Use the env variable:
```ts
const API_BASE = import.meta.env.VITE_API_URL || 'http://127.0.0.1:8000/api';
fetch(`${API_BASE}/auth/me/`, ...)
```

---

### 10. No JWT Token Auto-Refresh (UX / SECURITY)
**File:** `src/contexts/AuthContext.tsx`, `src/services/api.ts`
**Risk:** Access tokens expire after 1 day. No automatic refresh — users get silently logged out mid-session with no feedback.
**Fix:** In `apiRequest()`, on 401 response, attempt token refresh using `emsi_refresh` cookie before failing:
```ts
if (response.status === 401) {
  const refreshed = await tryRefreshToken();
  if (refreshed) return apiRequest(endpoint, options); // retry
  logout();
}
```

---

### 11. SSH Private Keys in db_config.json (SECURITY)
**File:** `backend/db_config.json`
**Risk:** SSH private keys and DB passwords stored in a JSON file. If this file is committed to git, credentials are exposed.
**Fix:** Verify `db_config.json` is in `.gitignore`. Move all secrets to environment variables.

---

### 12. print() Statements in Production Code (PERFORMANCE)
**File:** `backend/notifications/views.py`, `backend/forums/views.py`
**Risk:** Floods server logs, degrades performance under load.
**Fix:** Replace all `print()` with `logger.debug()` or remove entirely.

---

### 13. No Input Sanitization on Rich Text Content (SECURITY / XSS)
**Files:** `backend/forums/models.py`, `backend/assignments/models.py`
**Risk:** Forum posts and assignment descriptions are stored and returned as-is. If rendered as HTML on the frontend, XSS attacks are possible.
**Fix:** Either sanitize on save using `bleach`, or ensure the frontend always renders as plain text / safe markdown only.

---

### 14. Passwords in Bulk Import CSV (SECURITY)
**File:** `backend/users/views.py` — `BulkUserImportView`
**Risk:** Auto-generated passwords are never communicated to users. Imported users have no way to log in.
**Fix:** Send a welcome email with a password reset link to each created user, or return generated passwords in the import report (displayed once, never stored).

---

### 15. No Pagination Enforcement on Heavy Endpoints (PERFORMANCE)
**Files:** `backend/users/views.py`, `backend/audit_logs/views.py`, `backend/resources/views.py`
**Risk:** All records returned in a single response. With thousands of users/resources/logs, responses become huge and slow.
**Fix:**
```python
# settings.py
REST_FRAMEWORK = {
    'DEFAULT_PAGINATION_CLASS': 'rest_framework.pagination.PageNumberPagination',
    'PAGE_SIZE': 25,
}
```

---

## PERFORMANCE — Optimize for Scale

### 16. N+1 Query Problem in Serializers
**Files:** `backend/rooms/serializers.py`, `backend/resources/serializers.py`, `backend/quizzes/serializers.py`
**Risk:** Each serialized object triggers additional DB queries for counts and related objects.
**Fix:** Add `select_related` and `prefetch_related` to querysets:
```python
def get_queryset(self):
    return Room.objects.select_related('owner').prefetch_related('participants').annotate(
        participants_count=Count('participants'),
        resources_count=Count('resources'),
    )
```

---

### 17. Base64 Binary Files in JSON Responses (PERFORMANCE)
**Files:** `backend/users/serializers.py`, `backend/events/serializers.py`
**Risk:** Base64 encoding adds 33% size overhead. Large images/videos embedded in JSON responses cause slow API calls.
**Fix:** Serve binary files via dedicated endpoints (`/api/users/{id}/profile-picture/` already exists) and return only the URL in list responses, not the data.

---

### 18. No Server-Side Caching (PERFORMANCE)
**File:** `backend/platform_settings/views.py`
**Risk:** Platform settings are fetched from DB on every request. With many users, this is unnecessary DB load.
**Fix:**
```python
from django.core.cache import cache

def get(self, request):
    settings = cache.get('platform_settings')
    if not settings:
        settings = PlatformSettings.get_settings()
        cache.set('platform_settings', settings, timeout=300)  # 5 min
```

---

### 19. Polling Every 5 Minutes Per Client (PERFORMANCE)
**File:** `src/contexts/PlatformContext.tsx`
**Risk:** Every connected browser polls `/api/platform/public/` every 5 minutes. With 500 users, that's 100 requests/minute just for platform settings.
**Fix:** Increase interval to 30 minutes, or use a WebSocket/SSE for real-time settings updates.

---

### 20. No Database Indexes on Frequently Queried Fields (PERFORMANCE)
**Risk:** Fields like `Resource.status`, `Notification.recipient`, `AuditLog.action`, `AttendanceRecord.student` are filtered on frequently but have no DB indexes.
**Fix:** Add `db_index=True` to frequently filtered fields:
```python
status = models.CharField(max_length=10, choices=STATUS_CHOICES, db_index=True)
```
